# 八字解析————用AI让八字聊天

## 项目背景
### 关于八字：
我觉得命理师解读的过程抽象出来，很像一个RAG：命理师观察要素、去数据库检索相关意向信息、再重新组合成为他的解读

<img width="943" alt="image" src="https://github.com/user-attachments/assets/6f896d91-63f5-4bbc-b125-62cfa2daee7e" />


在网课自己学习时、和线下入门课见到一些新人，我发现这些数据库中的知识是最大的门槛，不懂的人见到名词就会觉得 “好厉害”、“名词好高深”（没办法仔细思考你说得对不对 听着玄乎，有的有道理）


2月份的时候，deepseek引起一股玄学热，但据我观察：
- 大部分人停留在看故事和自己对应的程度，无法深究；
- 由于反问和幻觉 llm可能偏离 但是提问人察觉不到（比如一开始排盘就排错了，所以建议直接输入四柱解读，而不是属于出生时间，很容易排错
  
我就觉得该用规则排盘还是用规则，在此基础正确地逐步向AI提问，可能是一个弥补背景知识gap的好办法。


### 关于呈现形式：
一直看到AI都是从聊天框流式输出，有时候会觉得太长了，即使根据markdown分了结构，还是觉得文字的层次感不够强，很难真正读进去、更容易只是看个热闹。

刚好八字中每个字都有每个独特的意向，有一种拟人感。且意向之间都有很强的作用力，类似于角色与角色之间的交互。它更接近于网状，而不是平面的单一的。

所以有了这个想法，**让每个字自己发言：自己是什么意向，是哪个方向，可能表示什么特点**，提供一种更立体的叙事、而不是平铺直叙。

这也是目前基于LLM与解决实际问题的差距，即**突破语言的局限、触达线性之外的结构的能力**

（个人认为背后LLM从memory层面是从高维理解很多信息的，只是sample后的语言表达力也有限）

## 针对用户
所以这个项目的理念就是：
给想要不止是看个热闹、对八字及运作规则进行一定了解、但是困于记不住基础知识没有继续的用户。

如果他想要用AI辅助他学习八字，他也可以一句一句地去问AI。

那么这个项目就把他即将一句句问的内容都提前内置，而且页面元素更丰富、仿佛自己的八字真的在互相交流，比只能对一个AI提问更丰富。

## 实现思路
一方面把初学者可能的提问提前写成prompt、填入个人信息即可调用AI；

另一方面把AI返回的内容**用工程重新解析，用更丰富的页面元素呈现出来**

AI调用逻辑：

固定prompt模板、要求AI直接返回对话形式的内容，但是。（说人话就是，让AI一次性装成好几个角色在对话...


丰富页面结构有：

1.八字角色（每个八字为一个角色，以聊天的形式呈现。感知上接近multi-agent的效果）

2.同党异党（塑造阵营：同我、生我者从聊天右侧也就是本人视角发言，泻克耗者从聊天左侧也就是对方的角度发言。接近常规对话思维。）

3.目前区分了基础（解读五行、基础位置规则）与进阶（十神、做功），通过两个固定prompt、两次调用达成，可以再加更多层级。

4.可以最后增加用户对话。

## 关于AI的实践感受
一次模拟multi-agent的尝试，明明只是一个AI，却想创造出更丰富的效果。

目前multi-agent的能力应该是比这个更低的，曾经看到一个说法“multi-agent就像让几个能力60%的P人协作，越协作效果越差”。

也是表达了我希望能够在AI能力之外，通过可扩展的设计性、工程性工作，给用户带来更好的特定任务使用体验。

## 关于八字的实践感受
在八字解读结合AI这个领域上，我能想到的从产品创新主要是以上层面了。

但是这个作品想要真正受到领域爱好者的认可，还应该内置很多的知识理念。因此这只能是一个tiny toy，达成了技术能触达的部分，还有很长的专业性之路要走。
