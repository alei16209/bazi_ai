# 八字解析————用AI让八字聊天
（由于私人网站域名尚在备案流程中，通过github形式呈现）

## 项目背景
关于八字：
我觉得命理师解读的过程抽象出来，很像一个RAG：命理师观察要素、去数据库检索相关意向信息、再重新组合成为他的解读

<img width="943" alt="image" src="https://github.com/user-attachments/assets/6f896d91-63f5-4bbc-b125-62cfa2daee7e" />


在网课自己学习时、和线下入门课见到一些新人，我发现这些数据库中的知识是最大的门槛，不懂的人见到名词就会觉得 “好厉害”、“名词好高深”（没办法仔细思考你说得对不对 听着玄乎，有的有道理）
2月份的时候，deepseek引起一股玄学热，但据我观察：大部分人停留在看故事和自己对应的程度，无法深究；由于反问和幻觉 llm可能偏离 但是提问人察觉不到；也有可能一开始排盘就排错了
我就觉得AI可能是一个弥补背景知识gap，能够让人关注到逻辑本身、而不是被名词劝退的好工具。

关于呈现形式：

一直看到AI都是从聊天框流式输出，有时候会觉得太长了，即使根据markdown分了结构，还是觉得文字的层次感不够强。

刚好有时研究八字，八字中每个字都有每个独特的意向，且意向之间都有很强的作用力，呈现接近于网状结构。

平常个人研究八字的时候就喜欢画结构之间的关系，而不是用纯流式文本去描述。所以有了这个想法，**让每个字自己发言：自己是什么意向，代表什么，遇到什么**。

这也是目前基于LLM与完全解决实际问题的差距，即**突破语言的局限、触达线性之外的结构的能力**（个人认为背后LLM从memory层面是从高维理解很多信息的，只是无法表达）

## 项目理念
所以这个项目的理念就是，固定prompt模板得到流式返回，但是**用工程重新解析，用更丰富的结构呈现出来**。（说人话就是，调一次AI，但是装成好几个角色在对话...

设计的结构有：

1.八字角色（每个八字为一个角色，以聊天的形式呈现。感知上接近multi-agent的效果）

2.同党异党（塑造阵营：同我、生我者从聊天右侧也就是本人视角发言，泻克耗者从聊天左侧也就是对方的角度发言。接近常规对话思维。）

3.目前区分了基础（解读五行、基础位置规则）与进阶（十神、做功），通过两个固定prompt、两次调用达成，可以再加更多层级。

4.可以最后增加用户对话。

## 碎碎念
在八字解读结合AI这个领域上，我能想到的从产品创新主要是以上层面了。

但是这个作品想要真正受到领域爱好者的认可，还应该内置很多的知识理念。因此这只能是一个tiny toy，达成了技术能触达的部分，还有很长的专业性之路要走。

此外，明确LLM能力边界，能用规则低成本解决的排盘、不用AI。
